================================================================================
1. Spatial Mapping and Distance Features
================================================================================

1.1 Point-in-polygon ward assignment
Used:
- Assign each cafe point to a ward polygon using geometric coverage logic.

Why used:
- This is the standard spatial way to map a point into an administrative boundary.

External sources:
- Shapely covers:
  https://shapely.readthedocs.io/en/2.1.2/reference/shapely.covers.html
- Shapely predicates overview:
  https://shapely.readthedocs.io/en/2.1.0/predicates.html

1.2 Nearest polygon fallback for unmatched points
Used:
- If a point was not covered by any ward polygon, assign the nearest ward polygon.

Why used:
- This is a practical geospatial fallback when boundary mismatch or geometry precision issues leave some points unmatched.

External sources:
- Shapely distance:
  https://shapely.readthedocs.io/en/2.1.0/reference/shapely.distance.html
- Shapely geometry manual:
  https://shapely.readthedocs.io/en/stable/manual.html

Note:
- The exact decision to use nearest-ward fallback is an adapted project rule, but it is based on standard geometric distance methodology.

1.3 Haversine-style radial distance for competitors and amenities
Used:
- Great-circle style point-to-point distance over latitude/longitude coordinates.

Why used:
- The datasets are stored in lat/lng and the needed features are radius-based counts and minimum distances.

External sources:
- GeoPy distance docs:
  https://geopy.readthedocs.io/en/stable/
- GeoPy PDF, calculating distance section:
  https://geopy.readthedocs.io/_/downloads/en/stable/pdf/

1.4 Amenity proximity counts
Used:
- Count locations within 500m and 200m, and compute minimum distance.

Why used:
- Radius-based neighborhood features are standard spatial accessibility features.

External sources:
- GeoPy distance docs:
  https://geopy.readthedocs.io/en/stable/
- Shapely manual:
  https://shapely.readthedocs.io/en/stable/manual.html

3. Fallback Feature Formulas Used During Combination
================================================================================

3.1 Road fallback estimation
Used:
- Estimated road-related features when exact road geometry was not directly usable in the local offline pipeline.

Why used:
- The current road raw file did not expose directly ready-to-use line geometry in the implemented script.
- A fallback estimate was needed to keep model features complete.

Method basis:
- clipped numeric transformations
- density-adjusted scoring
- bounded heuristics

External sources:
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html
- GeoPy distance docs:
  https://geopy.readthedocs.io/en/stable/

Note:
- The exact road fallback formula is an adapted composite heuristic.

3.2 Accessibility score
Used:
- Combined road distance, bus-stop density, school density, and hospital density into a bounded 0-10 score.

Why used:
- Accessibility scoring commonly combines transport access and nearby service density.

Method basis:
- bounded linear contribution from road distance
- additive access bonuses from key nearby services
- clipping to a fixed score range

External sources:
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html
- GeoPy distance docs:
  https://geopy.readthedocs.io/en/stable/

Note:
- The exact weights and caps are adapted project weights built from standard bounded-score design.

3.3 Foot-traffic score
Used:
- Combined density, transit, institutions, commerce, and road access into a bounded 0-10 score.

Why used:
- These variables are common proxies for urban activity intensity.

Method basis:
- additive signal construction
- bounded component caps
- clipping into a fixed interpretive score range

External sources:
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html
- Voting-style additive predictive combination example context:
  https://scikit-learn.org/stable/auto_examples/ensemble/plot_voting_regressor.html

Note:
- The exact component weights are adapted project weights.

3.4 Competition pressure score
Used:
- Combined nearby competitor counts, weighted pressure, and available quality signals into a bounded 0-10 score.

Why used:
- Competitive density is a standard business-location signal.

Method basis:
- weighted additive pressure score
- clipping to a bounded range

External sources:
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html
- scikit-learn RandomForestRegressor feature context for non-linear tabular relationships:
  https://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestRegressor.html

Note:
- The exact formula is an adapted composite heuristic.

7. Observed-Outcome Proxy Labeling
================================================================================

7.1 Percentile-based normalization
Used:
- 5th and 95th percentile clipping before min-max normalization.

Why used:
- Robust scaling to reduce the effect of outliers.

External sources:
- pandas DataFrame.quantile:
  https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.quantile.html
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html

7.2 observed_outcome_score
Used:
- weighted combination of normalized education intensity, normalized nearby students, normalized location density, normalized ward population, and a weak spatial pattern.

Why used:
- The local data needed a regression target with real spread.
- This proxy target was designed to use broader contextual signals instead of copying the pseudo-label formula exactly.

External sources:
- pandas quantile:
  https://pandas.pydata.org/docs/reference/api/pandas.DataFrame.quantile.html
- NumPy log1p:
  https://numpy.org/doc/1.19/reference/generated/numpy.log1p.html
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html

Note:
- The exact weighting and coordinate sinusoid term are adapted project heuristics.
- The external references justify the normalization and transformation methods used to construct the label.

7.3 observed_outcome_confidence
Used:
- weighted bounded confidence proxy based on normalized support signals.

Why used:
- A rough metadata signal for label confidence.

External sources:
- NumPy clip:
  https://numpy.org/doc/1.25/reference/generated/numpy.clip.html

Note:
- The exact formula is an adapted project heuristic.

7.4 Outcome tier conversion
Used:
- low_outcome if score < 40
- medium_outcome if score >= 40 and < 70
- high_outcome if score >= 70

Why used:
- Ordinal bucketing of a continuous score for easy interpretation.

External sources:
- pandas cut for interval-based binning context:
  https://pandas.pydata.org/docs/reference/api/pandas.cut.html

Note:
- The exact thresholds are adapted project thresholds.
